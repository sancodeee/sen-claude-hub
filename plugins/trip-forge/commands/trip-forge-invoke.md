你现在必须立即调用已安装的 `trip-forge` skill 处理用户当前的旅行规划需求。

用户需求：
$ARGUMENTS

严格执行 skill 的流程，内容规范读 `references/template.md`、设计模板读 `assets/report-template.html`，两者都不得凭记忆替换或简化：

1. 先检查目的地、天数、同行人、出发城市、出行时间、预算和旅行风格。
2. 缺失信息时一次性追问，不要自行假设或生成文件。
3. 信息完整后联网交叉核实天气、票价、营业时间、预约、交通班次和季节性开放，记录来源与核验日期。
4. 按维度分段写入文件，生成一份完整、自包含、移动端优先的 HTML 报告。
5. 不得省略六个维度、景点详情卡、看天调度、置信度、B 计划、来源附录或 HTML 设计要求；六维度同等完整，不压缩任何一维。
